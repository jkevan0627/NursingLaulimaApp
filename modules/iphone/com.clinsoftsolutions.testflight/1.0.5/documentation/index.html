testflightti Module

Description

iOS module for integration with the TestFlight SDK You need to download the TestFlight SDK seperately.

This way you (the developer) can control and upgrade the TestFlight SDK

Installation

See the Appcelerator Wiki for instructions on installing modules.

In order to use the module you also need to download the TestFlight SDK from the TestFlight website.

Then follow these steps:

  1. unzip the zip archive
  2. Copy the libTestFlight.a to /usr/local/lib

Accessing the testflightti Module

To access this module from JavaScript, you would do the following:

var testflightti = require("com.clinsoftsolutions.testflight");

The testflightti variable is a reference to the Module object.

Reference

testflightti.takeOff([teamToken], [testingFlag])

Initiate the Testflight SDK with the team token obtained from the TestFlight website. This is required and should be the first call to the module you make.

Your team token can be obtained from the TestFlight website at a href="https://testflightapp.com/dashboard/team/debugger/"

setOptions({options});

Set SDK options. Pass a javascript object. Possible keys are: - logToConsole: true - logToSTDERR: true - sendLogOnlyOnCrash: false - attachBacktraceToFeedback: false - disableInAppUpdates: false

All take bool values. Default values are shown

testException()

Raise an exception. Use to test crash reporting

passCheckpoint([checkpoint text])

Record that a checkpoint in the application has been passed. This will so log an INFO message on the console during debug sessions

launchFeedbackView()

Launch the SDK built in feedback window.

submitFeedback([feedback text])

Submit feedback text directly. This does not open the feedback window. So this can be used from your own custom form

remoteLog([log text])

Log some text remotely

addCustomEnvironmentInformation([key], [information])

Add custom environment information If you want to track custom information such as a user name from your application you can add it here

  • key: The key to store the information with
  • information: A string containing the environment you are storing

setTesting()

This sets the UDID in the testflight SDK. ONLY use in testing. Note that if you upload an app to Apple with this set then your app WILL BE REJECTED.

oldUDID

Return the old iOS UDID (This will not work on SDK >5.1 as apple have depreciated it). Do not use in production / release app. Apple will reject your binary

Exmaple Usage

var TF = require("com.clinsoftsolutions.testflight");
TF.takeOff('___ TEAM TOKEN ID ___');
TF.passCheckpoint("Application started");
